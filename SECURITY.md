# Security Policy

This repository is a desk-test prototype. Treat it as experimental hardware and software, not a production access-control system.

## Command Key

The committed command key is public sample data. Before flashing firmware or installing the iPhone app for real testing, generate a private 32-byte key:

```bash
openssl rand -hex 32
```

Paste the same generated bytes into both:

- `firmware/DoorUnlockerXiao/DoorUnlockerXiao.ino`
- `ios/DoorUnlockerApp/DoorUnlocker/DoorCommandAuthenticator.swift`

If the app source, built app, firmware flash, or command key is shared publicly, rotate the key and rebuild both sides.

## Responsible Use

Do not depend on this prototype for emergency egress, property security, or unattended access. Test on a desk first, then review the mechanical and safety implications before mounting anything to a real door.

Report security issues through a private GitHub security advisory if available on the repository, or open an issue with minimal sensitive detail.
