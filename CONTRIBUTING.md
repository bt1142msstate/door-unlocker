# Contributing

Contributions are welcome if they keep the project practical, documented, and safe to reproduce.

## Guidelines

- Do not commit private signing keys, Apple Developer Team IDs, provisioning profiles, or device-specific secrets.
- Keep hardware changes clear enough for a desk-test builder to follow.
- Include screenshots or wiring diagrams when changing the physical layout.
- For iOS changes, verify the app still builds in Xcode.
- For firmware changes, include the target board and library versions you used.

## Pull Requests

Small, focused pull requests are easiest to review. Include what changed, why it changed, and how you tested it.
